Signals
Created 2019-06-20| Updated 2019-06-29

Signals are software generated interrupts of simply put a way that a proecss’s state is recognised by a parent process mostly the latter being the Operating System.

When a signal is recieved by a program it stops whatever it is doing and handles the signal or either avoids but acknowledges that it recieved the signal somehow.

Signals are mostly used for runtime management of the programs as the only way CPU can communicate or control the state of the program is by using signals.

Simple Example For Signal Handling

Here are all the signals that are defined in a standard Linux machine each with its own distinct integral value.

Common Signals

SIGINT - Interrupt (mostly raised by CTRL+C)
SIGTERM - Terminate the process
SIGILL - Illegal Instruction
SIGSTOP - Process stops execution (CTRL+Z)
